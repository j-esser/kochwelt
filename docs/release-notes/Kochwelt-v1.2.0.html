Kochwelt 1.2.0

Persönliche Rezeptverwaltung & Wochenplaner — Update Mai 2026

Liebe Tester,

danke fürs Mittesten der Vorgänger-Version! 1.2.0 bringt vor allem eines: bessere Auffindbarkeit versteckter Funktionen. Ein neues Tipp-System weist auf nützliche Kniffe hin, die bisher unterzugehen drohten — etwa die clevere iOS-Funktion zum Scannen von Texten direkt aus dem Kochbuch. Außerdem könnt ihr nun einzelne Rezepte gezielt teilen, die Standard-Portionen für den Planer einstellen, und ich habe zwei nervige Bugs aus 1.1.0 behoben.

Was ist neu in 1.2.0

Tipps & Tricks — Hilfe direkt im Kontext

Auf jedem Bildschirm findet ihr jetzt ein kleines Fragezeichen-Symbol ❓. Antippen öffnet einen Hinweis-Dialog mit den Funktionen, die in diesem Bereich oft übersehen werden. Eine zentrale Sammlung aller Tipps gibt es zusätzlich unter Einstellungen → Tipps & Tricks, gruppiert nach Bereich und mit aufklappbaren Beschreibungen.

Highlight: Im Eingabefeld für Zutaten oder Zubereitung weist die App jetzt darauf hin, dass iPhone-Nutzer mit einem Doppel-Tipp ins Feld die Option „Text scannen" bekommen — die Kamera erkennt dann Text aus einer Seite oder einem Zettel und fügt ihn direkt ein. Sehr praktisch beim Übernehmen aus Kochbüchern.

Einzelnes Rezept als JSON teilen

In der Rezept-Detailansicht oben rechts gibt es ein neues Teilen-Symbol. Tippt darauf, um das Rezept als kompakte .json-Datei per Mail, Nachrichten, AirDrop oder in die Dateien-App zu schicken. Die andere Person importiert die Datei einfach unter Rezepte → Tools → Importieren — bestehende Rezepte werden aktualisiert, neue hinzugefügt, vorhandene Fotos bleiben erhalten.

Standard-Portionen für den Wochenplaner

Unter Einstellungen → Wochenplaner könnt ihr einstellen, mit wie vielen Portionen eine Mahlzeit beim Hinzufügen zum Plan vorbelegt sein soll (Default: 2). Die Einstellung wirkt im Rezept-Picker, im „Zum Planer"-Dialog und im Snack-/Kalte-Küche-Dialog.

Erinnerungen — neuer Default

Bei einer Neuinstallation steht die Erinnerung jetzt auf Wöchentlich (Montag, 8 Uhr) statt auf „Nie". Wer das bereits manuell gesetzt hatte, behält seine Einstellung — neu ist nur der Startwert.

Bugfixes

  • Neues Rezept wurde manchmal nicht gespeichert. Ursache: Das vorgeschlagene Standard-Foto verursachte einen unbemerkten Fehler beim Speicher-Vorgang, das Rezept landete dann einfach nicht in der Liste. Behoben.
  • Hochkant-Fotos (z. B. vom iPhone) sprengten manchmal das Karten-Layout in der Rezeptliste — das Foto wurde extrem hoch dargestellt. Karten haben jetzt eine feste Bildhöhe und beschneiden den Ausschnitt sauber.

Was ihr gerne testen dürft

  • Tipp-Symbole entdecken: Auf jedem Bildschirm das ❓-Icon antippen — sind die Hinweise nützlich? Fehlt euch ein Tipp?
  • iOS-Text-Scan: Bei „Neues Rezept" doppelt ins Zutaten-Feld tippen → „Text scannen" → Kochbuchseite scannen. Wie zuverlässig ist die Erkennung?
  • Rezept teilen: In der Detailansicht das Teilen-Symbol nutzen. Auf einem zweiten Gerät (oder im Mail-Postfach) die Datei öffnen → in Kochwelt importieren → kommt das Rezept sauber an?
  • Wochenplaner-Default: Standard-Portionen in den Einstellungen z. B. auf 4 setzen, dann eine Mahlzeit zum Plan hinzufügen — ist die Vorbelegung dort?
  • Eigenes Hochkant-Foto als Rezept-Foto auswählen — wirkt die Karte in der Rezeptliste sauber?
  • Speichern-Bug: Neues Rezept anlegen, ohne ein eigenes Foto zu wählen → das Default-Foto bleibt → speichern. Erscheint das Rezept in der Liste?
